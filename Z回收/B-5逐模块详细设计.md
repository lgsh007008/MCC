**模块1：每日加工登记（阶段一降级为可选）**

> **阶段一调整说明（2026-05-23）**：工资计算实际基于销售方量而非加工方量，模块1在阶段一不参与毛利核算。父母自记加工单也容易漏记，录入流程不优先走通。阶段二启用，用于计算出材率、优化采购和排产。

核心发现：成品单上记录的是方数(m)，不是吨。工资不由系统按人计算，而是由领班线下分配。

工资机制：

- 生产总工资 = 当月**销售**总方数 × 165元/方（所有规格统一价，**非加工方量**）
- 领班拿到总工资后，按工人上工天数分配给每人（系统不涉及）
- 修锯师傅：固定3100元/月，单独核算
- 次品扣减：发货后客户不满意才算次品，扣减总方数
- 发放周期：一季度一发（收麦、收秋、过年），但月度核算应按月计提
- 借支：偶尔有，频次低

**模块2：销售登记与欠款管理（基于原设计微调）**

核心发现：

- 销售单据上有多种规格明细，每种规格独立记录方量和金额
- 规格格式：厚×宽×长×根数，方量 = 厚×宽×长×根数（自动计算）
- 单价按元/m³，不同客户不同单价
- 欠款是常态：客户不一定当场结清，有累计欠款
- 欠款追踪方式：每次新销售在原欠款上相加，回款在原欠款上相减
- **正规化新增**：付款账户字段（对公/个人），支持资金流合规追踪

**模块3：采购登记（基于采购调研设计）**

调研发现：

- 供应商大部分固定，也有随机
- 采购方式：送货上门为主，偶尔自提（付运费）
- 定价方式：按吨计价
- 付款方式：对公/微信/现金，大部分现金，偶有欠款
- 有书面单据
- 质检：看虫眼等，折价收（如1.5吨按1吨计）
- 无固定分区，无批次概念
- 库存感知：原木大概知道，每季度盘点（吨×单价）

供应商表见模块0（表0-2）。

| 备注       | 文本       | 否    | 送货上门/自提、运费等          |

折价处理说明：

- 送货吨数 ≠ 结算吨数时（如1.5吨按1吨计），两个字段分别记录
- 采购金额按结算吨数计算

欠款追踪逻辑（与销售模块对称）：

- 应付金额 = 采购金额 - 已付金额，一眼看出每笔采购还有多少待付
- 应付金额 > 0 → 未付清；应付金额 = 0 → 已付清
- 供应商表的累计欠款自动汇总，一眼看出总共欠某个供应商多少

MVP阶段不做的：

- 批次管理（原木按规格拣选，无批次概念）
- 复杂质检流程（简单标记即可）

**模块4：月度报表**

**表4-1：毛利核算表**

| 项目        | 金额      | 计算方式                     |
| --------- | ------- | ------------------------ |
| 销售收入（出货）  | xxx     | SUM当月销售单据的合计金额           |
| 减：采购支出    | xxx     | SUM当月采购记录的采购金额           |
| 减：计件工资    | xxx     | 当月销售总方数×165 + 3100（修锯师傅） |
| **月度毛利**  | **xxx** | 销售收入 - 采购支出 - 计件工资       |
| 毛利率       | xx%     | 月度毛利 / 销售收入              |
| ---       | ---     | ---                      |
| 期末库存调整    | +/-xxx  | 季度盘点后手动填写                |
| **调整后毛利** | **xxx** | 月度毛利 + 库存调整              |

库存变动调整说明（MVP阶段）：

- 采购旺季毛利可能偏低（库存积压）
- 销售旺季毛利可能偏高（消耗库存）
- 季度盘点时：盘点吨数 × 平均采购单价 = 期末库存成本
- 手动调整后更准确
