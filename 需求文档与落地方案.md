# 木材厂经营项目 - 需求文档与落地方案

## 一、项目概述

### 项目目标

将家中小木材厂的业务和财务从当前无序状态，系统化、数字化，最终通过飞书多维表格搭建数字化系统承载。

### 方法论资产（全阶段适用）

全阶段参考之前创业做SHOPEE的全流程经验，包括：

- **说明解释体系**：SOP文档、操作说明、业务解释框架
- **事项体系**：任务拆解、进度追踪、优先级管理
- **全流程应用**：用飞书多维表格搭建的完整业务流程系统

**阶段一：线上化 —— 打地基 + 先看见**

| 项目       | 内容                               |
| -------- | -------------------------------- |
| **目标**   | 业务流程初步线上化，算出月度毛利润                |
| **原则**   | 业务记录不优化、不改动、不改流程，粗颗粒度也行          |
| **做什么**  | 把纸笔记录搬到系统：采购、加工、销售、回款、工资，原样录入    |
| **关键产出** | 月度毛利报表（经营线+现金流线）                 |
| **验收标准** | 连续3个月毛利数据与实际资金进出基本吻合             |
| **核心公式** | 月度毛利 = 当月销售总额 - 当月采购总支出 - 当月工资总额 |
| **技术选型** | 飞书多维表格                           |

选型理由：

- 专业MTO模板违背"不优化不改动"原则，强制填标准字段会遭线下抵制
- 新低代码平台学习成本高，分散精力拉长周期
- 飞书多维表格能力溢出（关联、引用、自动化、仪表盘、API），支撑5人小厂绰绰有余
- 微信小程序直接填报，中年用户接受度最高
- 未来迁移不是问题：跑通飞书后再上专业ERP是带着实战数据模型迁移，成功率远高于现在裸上

## 二、业务现状分析

### 业务特征

| 特征   | 现状              |
| ---- | --------------- |
| 生产模式 | 以销定产，有订单才生产     |
| 库存管理 | 无库存概念，无批次概念     |
| 加工方式 | 根据订单要求，随机选择原木加工 |

### 收入结构

| 类型   | 说明     | 管理现状          |
| ---- | ------ | ------------- |
| 主要收入 | 板材销售   | 有单据记录，有客户欠款追踪 |
| 次要收入 | 加工废料出售 | 待确认           |

### 支出结构

| 类型   | 说明           | 管理现状        |
| ---- | ------------ | ----------- |
| 主要支出 | 采购原木         | 无采购单、无采购账本  |
| 日常开支 | 待梳理          | 无记账         |
| 工人工资 | 按方数计算，每人单价不同 | 有每日加工记录作为依据 |

### 客户情况

- 当前5-6家活跃客户（以前30-40家，流失严重）
- 客户持续流失（外地木材更便宜）
- 收款以账期形式
- 对账习惯：客户付款时对账，无定期对账

### 当前已有记录

1. **每日加工记录**：加工完成数量及方数（用于计算员工工资）
2. **销售订单记录**：每位客户的出货单据、金额、欠款总额（回款即减）

### 核心痛点

1. **采购完全无序**：无采购单、无采购账本，成本失控
2. **客户持续流失**：外地竞争+账期收款，现金流压力大
3. **财务黑洞**：支出无记账，收入支出无法对账
4. **无库存管理**：原木和板材均无库存概念，无法核算成本

| --- | --------- | --- | ------------------------------------------ |
| 模块0 | 基础数据      | P0  | 供应商表、客户表                                   |

| 模块4 | 月度报表      | P0  | 毛利核算表、现金流表（含其他收入/其他支出）                     |

***

**B-2：核心公式**

```
当月计件工资 = 当月销售总方数 × 165元/方（工资按销售方量算，不按加工方量）
当月工资总额 = 当月计件工资 + 3100（修锯师傅）
月度毛利 = 当月销售总额 - 当月采购总支出 - 当月工资总额
毛利率 = 月度毛利 / 当月销售总额
```

**库存变动说明（MVP阶段简化处理）**：

- 采购的原木 ≠ 立即消耗的原木，存在库存周期
- MVP阶段用「当月采购额」代替「当月消耗原木成本」计算毛利，会导致：
  - 采购旺季毛利偏低（库存积压）
  - 销售旺季毛利偏高（消耗库存）
- **季度盘点时**手动调整：期末库存原木折算成本 = 盘点吨数 × 平均采购单价
- 调整后公式：`实际毛利 = 月度毛利 + 库存增加（或 - 库存减少）`

***

**B-3：核算口径**

| 维度   | 取数方式            | 用途            |
| ---- | --------------- | ------------- |
| 经营核算 | 出货金额（权责发生制）     | 看经营能力，判断降价空间  |
| 现金流  | 实际回款金额 - 实际付款金额 | 看资金健康度，判断账期风险 |

***

**B-4：不做的清单**

- 单笔订单毛利核算（原木随机匹配，无法归集）
- 批次管理、原木追溯
- 实时库存管理
- 采购审批流程
- 客户流失预警、产品目录标准化
- 员工主数据管理（5人固定，无临时工，领班线下分配）
- 规格体系标准化（规格作为文本字段自由录入）

**模块1：每日加工登记（阶段一降级为可选）**

> **阶段一调整说明（2026-05-23）**：工资计算实际基于销售方量而非加工方量，模块1在阶段一不参与毛利核算。父母自记加工单也容易漏记，录入流程不优先走通。阶段二启用，用于计算出材率、优化采购和排产。

核心发现：成品单上记录的是方数(m)，不是吨。工资不由系统按人计算，而是由领班线下分配。

工资机制：

- 生产总工资 = 当月**销售**总方数 × 165元/方（所有规格统一价，**非加工方量**）
- 领班拿到总工资后，按工人上工天数分配给每人（系统不涉及）
- 修锯师傅：固定3100元/月，单独核算
- 次品扣减：发货后客户不满意才算次品，扣减总方数
- 发放周期：一季度一发（收麦、收秋、过年），但月度核算应按月计提
- 借支：偶尔有，频次低

**模块3：采购登记（基于采购调研设计）**

调研发现：

- 供应商大部分固定，也有随机
- 采购方式：送货上门为主，偶尔自提（付运费）

折价处理说明：

- 送货吨数 ≠ 结算吨数时（如1.5吨按1吨计），两个字段分别记录
- 采购金额按结算吨数计算

欠款追踪逻辑（与销售模块对称）：

- 应付金额 = 采购金额 - 已付金额，一眼看出每笔采购还有多少待付
- 应付金额 > 0 → 未付清；应付金额 = 0 → 已付清
- 供应商表的累计欠款自动汇总，一眼看出总共欠某个供应商多少
