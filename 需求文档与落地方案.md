# 木材厂经营项目 - 需求文档与落地方案

***

## 一、项目概述

### 项目目标

将家中小木材厂的业务和财务从当前无序状态，系统化、数字化，最终通过飞书多维表格搭建数字化系统承载。

### 方法论资产（全阶段适用）

全阶段参考之前创业做SHOPEE的全流程经验，包括：

- **说明解释体系**：SOP文档、操作说明、业务解释框架
- **事项体系**：任务拆解、进度追踪、优先级管理
- **全流程应用**：用飞书多维表格搭建的完整业务流程系统

### 版本规划

**阶段一：线上化 —— 打地基 + 先看见**

| 项目       | 内容                               |
| -------- | -------------------------------- |
| **目标**   | 业务流程初步线上化，算出月度毛利润                |
| **原则**   | 业务记录不优化、不改动、不改流程，粗颗粒度也行          |
| **做什么**  | 把纸笔记录搬到系统：采购、加工、销售、回款、工资，原样录入    |
| **关键产出** | 月度毛利报表（经营线+现金流线）                 |
| **验收标准** | 连续3个月毛利数据与实际资金进出基本吻合             |
| **核心公式** | 月度毛利 = 当月销售总额 - 当月采购总支出 - 当月工资总额 |
| **技术选型** | 飞书多维表格                           |

选型理由：

- 专业MTO模板违背"不优化不改动"原则，强制填标准字段会遭线下抵制
- 新低代码平台学习成本高，分散精力拉长周期
- 飞书多维表格能力溢出（关联、引用、自动化、仪表盘、API），支撑5人小厂绰绰有余
- 微信小程序直接填报，中年用户接受度最高
- 未来迁移不是问题：跑通飞书后再上专业ERP是带着实战数据模型迁移，成功率远高于现在裸上

**阶段二：精细化 —— 再理解**

| 项目       | 内容                                                |
| -------- | ------------------------------------------------- |
| **目标**   | 全面精细化梳理业务流程，发现降本增效的点，为公司正规化铺路                     |
| **原则**   | 只做细微补充，不推翻重来                                      |
| **做什么**  | 基于阶段一积累的真实数据，分析成本结构、利润分布、客户贡献、流程瓶颈；推进公司正规化的各项准备工作 |
| **关键产出** | 降本增效建议清单（按优先级排序，附预期收益）                            |
| **前提条件** | 阶段一数据已连续3个月可信                                     |

**阶段三：落地执行 —— 真改变**

| 项目       | 内容                                                             |
| -------- | -------------------------------------------------------------- |
| **目标**   | 将降本增效点付诸实施，真正降本增效；把正规化变成真实结果；明确父母角色并安排好父母财务                    |
| **原则**   | 逐项落地，每项验证效果                                                    |
| **做什么**  | 执行阶段二的建议清单；大姐加入作为实际业务助手，推动线下执行和正规化落地；明确父母在经营中的角色定位，做好父母的财务保障安排 |
| **关键产出** | 优化措施落地、成本下降、效率提升（用数据验证）；公司正规化落地；父母角色与财务安排落地                    |
| **人力变化** | 大姐加入，承担日常录入和业务协调；父母角色逐步调整                                      |

**阶段四：规模增长 —— 做大**

| 项目       | 内容                      |
| -------- | ----------------------- |

**总体路线：看见 → 理解 → 优化 → 增长**

1. 把现有业务原样搬上线，先看见每月到底赚了多少钱
2. 深挖业务细节，精细化流程找降本增效点，同时为公司正规化铺路
3. 让大姐加入落地执行，把优化和正规化变成真实结果，安顿好父母的角色和财务
4. 内部跑通了再向外扩张，增大工厂规模，借助SHOPEE经验加速增长

## 二、业务现状分析

### 业务特征

| 特征   | 现状              |
| ---- | --------------- |
| 生产模式 | 以销定产，有订单才生产     |
| 库存管理 | 无库存概念，无批次概念     |
| 加工方式 | 根据订单要求，随机选择原木加工 |

### 收入结构

| 类型   | 说明     | 管理现状          |
| ---- | ------ | ------------- |
| 主要收入 | 板材销售   | 有单据记录，有客户欠款追踪 |
| 次要收入 | 加工废料出售 | 待确认           |

### 支出结构

| 类型   | 说明           | 管理现状        |
| ---- | ------------ | ----------- |
| 主要支出 | 采购原木         | 无采购单、无采购账本  |
| 日常开支 | 待梳理          | 无记账         |
| 工人工资 | 按方数计算，每人单价不同 | 有每日加工记录作为依据 |

### 客户情况

- 当前5-6家活跃客户（以前30-40家，流失严重）
- 客户持续流失（外地木材更便宜）
- 收款以账期形式
- 对账习惯：客户付款时对账，无定期对账

### 当前已有记录

1. **每日加工记录**：加工完成数量及方数（用于计算员工工资）
2. **销售订单记录**：每位客户的出货单据、金额、欠款总额（回款即减）

### 核心痛点

1. **采购完全无序**：无采购单、无采购账本，成本失控
2. **客户持续流失**：外地竞争+账期收款，现金流压力大
3. **财务黑洞**：支出无记账，收入支出无法对账
4. **无库存管理**：原木和板材均无库存概念，无法核算成本

## 三、需求梳理

### 第一阶段设计

基于所有调研结果，设计完整的飞书多维表格系统。

***

**B-1：模块总览**

| 功能        | 说明                             |
| --------- | ------------------------------ |
| 供应商       | 维护原木供应商基础资料，用于关联采购单并统计各供应商采购情况 |
| 客户        | 维护客户基础资料，用于关联销售单、收款记录和客户欠款对账   |
| 采购单       | 记录每次原木采购的业务事实                  |
| 采购付款记录    | 记录每笔对供应商的付款，用于核对应付和统计实际现金流出    |
| 成品销售单     | 记录每次对客户的出货主单信息，包括客户、日期、总金额等    |
| 成品销售明细    | 逐行记录销售规格、根数、方量、单价等明细，用于销售核算    |
| 销售收款记录    | 记录每笔客户回款，用于核对应收和统计实际现金流入       |
| 财务事件类型字典表 | 统一维护财务收支分类口径，规范其他收入/支出的录入      |
| 财务事实事件表   | 汇总所有实际发生的财务收支事件，作为现金流统计的基础数据   |
| 财务结果汇总表   | 按月汇总销售、采购、工资及其他收支，输出毛利和现金流结果   |

***

**B-2：核心公式**

```
当月计件工资 = 当月销售总方数 × 165元/方（工资按销售方量算，不按加工方量）
当月工资总额 = 当月计件工资 + 3100（修锯师傅）
月度毛利 = 当月销售总额 - 当月采购总支出 - 当月工资总额
毛利率 = 月度毛利 / 当月销售总额
```

**库存变动说明（MVP阶段简化处理）**：

- 采购的原木 ≠ 立即消耗的原木，存在库存周期
- MVP阶段用「当月采购额」代替「当月消耗原木成本」计算毛利，会导致：
  - 采购旺季毛利偏低（库存积压）
  - 销售旺季毛利偏高（消耗库存）
- **季度盘点时**手动调整：期末库存原木折算成本 = 盘点吨数 × 平均采购单价
- 调整后公式：`实际毛利 = 月度毛利 + 库存增加（或 - 库存减少）`

***

**B-3：核算口径（不太懂二者的维度）**

| 维度   | 取数方式            | 用途            |
| ---- | --------------- | ------------- |
| 经营核算 | 出货金额（权责发生制）     | 看经营能力，判断降价空间  |
| 现金流  | 实际回款金额 - 实际付款金额 | 看资金健康度，判断账期风险 |

***

**B-4：销售录入与对账规则**

**数据来源原则：**

- 线下客户销售单是销售数据的唯一准确来源
- 系统中的销售单、销售明细、客户信息，均以线下单据为准进行录入和校验

**与线下账本对账（必须做）：**

1. **查新增客户**：线下销售单中是否出现了系统客户表中没有的新客户名，以确认是否存在整个客户都被漏掉的情况（线下有销售记录，但系统中完全没有该客户和对应单据）
2. **查遗漏单据**：从客户维度逐家核对，该客户在线下是否有销售单没录入系统（同一客户可能漏单）

**销售单规格录入口径（已确认 99%）：**

- **字段顺序**：`长度 × 宽度 × 厚度 × 根数`（固定，不变）
- **长度单位**：部分以米为单位（如 `2`、`2.6`、`4.5`，录入时 ×100 转为 cm），短料直接以 cm 为单位（如 `90`、`95`、`110`）
- **宽度单位**：cm
- **厚度单位**：cm（手写常省略小数点，如 `25`=2.5cm、`45`=4.5cm，需结合右侧方量反推确认）
- 详细规则见操作规范.md「规格推演与验算规则」

***

**B-5：开票对账规则（待定）**

> 公司为一般纳税人，客户要求开 13% 专票。开票是税务层面的操作，需与销售单、销售收款记录建立关联，避免出现"有票无单"或"有单无票"的盲区。

**待确认事项：**

- 开票时机：发货时开 / 收款时开 / 客户要求时开？
- 开票信息字段：发票号码、开票日期、开票金额、是否已开票标记
- 对账规则：如何确保每张票都有对应的销售单和收款记录？
- 历史未开票的销售单如何处理？

***

**B-6：销售收款对账规则（待定）**

**参考 B-4 销售录入对账规则，需确认：**

- 数据来源：线下收款流水的唯一准确来源是什么？（对公银行流水？微信账单？）
- 对账维度：按客户维度逐家核对收款金额 vs 系统记录？
- 漏记检查：线下已到账但系统中未录入的收款如何排查？
- 不同的产品对账形式也不同。
